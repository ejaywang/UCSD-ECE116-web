Assignment 1: ESP32-S3 Setup Guide

This guide will help you set up your development environment for Assignment 1 of the Embedded AI course. Follow these steps carefully to ensure your ESP32-S3-EYE board is properly configured for TensorFlow Lite Micro development.

Prerequisites

Before starting, ensure you have: - ESP32-S3-EYE Development Board - Micro-USB to USB-A cable - Computer with Windows, macOS, or Linux - Stable internet connection for downloading software

Step 1: Install Visual Studio Code

  1. Download and install Visual Studio Code from https://code.visualstudio.com/
  2. Launch VS Code after installation

Step 2: Install PlatformIO IDE Extension

  1. In VS Code, click on the Extensions icon in the left sidebar (or press Ctrl+Shift+X)
  2. Search for "PlatformIO IDE"
  3. Click "Install" on the official PlatformIO IDE extension by PlatformIO
  4. Wait for the installation to complete (this may take several minutes)
  5. Restart VS Code when prompted

Step 3: Verify PlatformIO Installation

  1. After restarting VS Code, you should see a PlatformIO icon in the left sidebar
  2. Click on the PlatformIO icon
  3. You should see the PlatformIO Home page open

Step 4: Create Your First ESP32-S3 Project

  1. In PlatformIO Home, click "New Project"
  2. Fill in the project details:
  3. Name: esp32_test
  4. Board: Search for and select "ESP32-S3-DevKitC-1" (this is compatible with ESP32-S3-EYE)
  5. Framework: Select "Arduino"
  6. Location: Use default or choose your preferred location
  7. Click "Finish"
  8. Wait for the project to be created and dependencies to be downloaded

Step 5: Connect Your ESP32-S3-EYE Board

  1. Connect your ESP32-S3-EYE to your computer using the micro-USB cable
  2. The board should power on (you may see LEDs light up)

Finding the Correct Serial Port

Windows: - Open Device Manager - Look under "Ports (COM & LPT)" for a device like "Silicon Labs CP210x USB to UART Bridge" or similar - Note the COM port number (e.g., COM3, COM4)

macOS: - Open Terminal - Run: ls /dev/cu.* - Look for a device like /dev/cu.usbserial-* or /dev/cu.SLAB_USBtoUART

Linux: - Open Terminal - Run: ls /dev/ttyUSB* or dmesg | grep tty - Look for a device like /dev/ttyUSB0

Step 6: Test Basic Connectivity

  1. In your esp32_test project, open src/main.cpp
  2. Replace the contents with this simple test code:

```cpp

include

void setup() { Serial.begin(115200); pinMode(LED_BUILTIN, OUTPUT); Serial.println("ESP32-S3 Test - Hello World!"); }

void loop() { digitalWrite(LED_BUILTIN, HIGH); Serial.println("LED ON"); delay(1000);

digitalWrite(LED_BUILTIN, LOW); Serial.println("LED OFF"); delay(1000); } ```

  1. Save the file (Ctrl+S)

Step 7: Upload and Test

  1. In the PlatformIO toolbar at the bottom of VS Code, click the "Upload" button (arrow pointing right)
  2. Wait for the code to compile and upload
  3. If successful, you should see "SUCCESS" in the terminal

Troubleshooting Upload Issues

If upload fails: 1. Check the cable: Ensure you're using a data cable, not just a charging cable 2. Try holding BOOT button: Hold the BOOT button on the ESP32-S3-EYE while clicking upload, release after upload starts 3. Check port selection: In platformio.ini, add the line upload_port = YOUR_PORT (replace with your actual port) 4. Driver issues: You may need to install CP210x drivers from Silicon Labs

Step 8: Monitor Serial Output

  1. Click the "Serial Monitor" button in the PlatformIO toolbar (plug icon)
  2. Set the baud rate to 115200
  3. You should see "ESP32-S3 Test - Hello World!" followed by alternating "LED ON" and "LED OFF" messages
  4. The onboard LED should blink every second

Step 9: Install Python Dependencies

Open a terminal/command prompt and install the required Python libraries:

bash pip install pyserial matplotlib numpy

Verification Checklist

Before proceeding to Assignment 1, ensure: - [ ] VS Code and PlatformIO are installed and working - [ ] You can create a new ESP32-S3 project - [ ] Your ESP32-S3-EYE board is recognized by your computer - [ ] You can successfully upload code to the board - [ ] You can see serial output in the Serial Monitor - [ ] The onboard LED blinks as expected - [ ] Python dependencies are installed

Common Issues and Solutions

Issue: "Device not found" or upload fails

Solution: - Check USB cable (must be data cable) - Hold BOOT button during upload - Install CP210x drivers - Try a different USB port

Issue: PlatformIO not appearing in VS Code

Solution: - Restart VS Code - Check if extension is properly installed - Try reinstalling the PlatformIO extension

Issue: Serial Monitor shows garbled text

Solution: - Check baud rate is set to 115200 - Ensure correct port is selected - Try disconnecting and reconnecting the board

Issue: Python libraries won't install

Solution: - Ensure Python is properly installed - Try using pip3 instead of pip - On some systems, use python -m pip install instead
