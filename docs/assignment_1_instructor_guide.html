ECE 4xx: Embedded Artificial Intelligence - Assignment 1 Instructor Guide

This document provides a detailed grading rubric and an automation strategy for Assignment 1: "Fundamentals of On-Device Inference."

1. Grading Rubric

Total Points: 100

| Category | Task | Points | Grading Method | |---|---|---|---| | ESP32-S3 Firmware | Task 1: Environment Setup | 15 | Automated | | | Task 2: Sensor Data Acquisition | 20 | Automated | | | Task 3: Model Integration & Inference | 30 | Automated | | Host-Side Script | Task 4: Data Visualization | 25 | Automated & Manual | | Documentation | Task 5: Report & Video | 10 | Manual |

Detailed Breakdown:

ESP32-S3 Firmware (65 points)

  • Task 1: Environment Setup (15 points)

    • [5 pts] PlatformIO project compiles without errors.
    • [10 pts] Basic sketch uploads and runs on the ESP32-S3 (verified by serial output).

  • Task 2: Sensor Data Acquisition (20 points)

    • [10 pts] Correctly reads analog sensor data.
    • [10 pts] Prints raw sensor data to serial monitor in the specified format.

  • Task 3: Model Integration & Inference (30 points)

    • [10 pts] TensorFlow Lite Micro model is correctly integrated.
    • [10 pts] Inference is performed on sensor data.
    • [10 pts] Both sensor data and inference result are printed to serial in the correct format.

Host-Side Script (25 points)

  • Task 4: Data Visualization
    • [10 pts] Python script runs without errors.
    • [10 pts] Correctly parses serial data.
    • [5 pts] Real-time plot updates dynamically (manual check of video).

Documentation (10 points)

  • Task 5: Report & Video
    • [5 pts] Report is clear, complete, and technically accurate.
    • [5 pts] Video demonstration clearly shows the working system.

2. Automation Strategy

Automated Grading Workflow:

  1. CI/CD Pipeline (GitHub Actions / GitLab CI):

    • Trigger: On student submission (e.g., push to a specific branch).
    • Steps:
      1. Checkout Code: Clones the student's repository.
      2. Compile Firmware: Uses PlatformIO CLI to compile the ESP32-S3 firmware.
      3. Run Unit Tests (Optional but Recommended): If unit tests are provided for the firmware, run them.
      4. Run Python Script: Execute the host-side Python script with a simulated serial port to check for errors and correct data parsing.

  2. Custom Python Grading Script:

    • Purpose: To perform more detailed checks on the serial output and functionality.
    • Features:
      • Serial Port Mocking: Use a library like pyserial-sim to simulate the ESP32-S3's serial output without needing physical hardware.
      • Data Format Validation: Check if the serial output adheres to the specified format (e.g., Analog: XXX, Inference: Y).
      • Inference Verification: For a given set of simulated sensor inputs, verify that the inference output from the ESP32-S3 matches the expected output within a tolerance.
      • Report Generation: The script can generate a report of the automated checks, which can be used by the instructor for final grading.

Manual Grading Workflow:

  1. Code Review:

    • Review the ESP32-S3 firmware and Python script for code quality, comments, and logic.
    • Check for plagiarism.

  2. Report and Video Review:

    • Read the student's report for clarity, completeness, and technical accuracy.
    • Watch the video demonstration to visually confirm the working system.

  3. Final Grade Calculation:

    • Combine the automated grading results with the manual review to determine the final grade.

3. Instructor Notes

  • Provide a Clear Model: The pre-trained TensorFlow Lite Micro model (model.h) should be simple and well-documented. Provide a clear explanation of its input and output format.
  • Serial Output Format: Emphasize the importance of adhering to the specified serial output format, as this is crucial for automated grading.
  • Common Issues to Watch For:
    • Incorrect PlatformIO project setup.
    • Incorrect wiring of the photoresistor circuit.
    • Issues with serial port permissions on the host PC.
    • Incorrect parsing of serial data in the Python script.
    • Blocking code in the main loop that prevents real-time updates.

By combining automated checks with manual review, this grading strategy provides a balance between efficiency and thoroughness, allowing for consistent and fair evaluation of student work while minimizing the instructor's grading workload.
